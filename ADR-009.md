ADR-009: Eliminación de columna 'synced' en Drift

Status: Aceptado

Contexto
- Durante migración y refactor, la columna 'synced' en HabitsTable y HabitLogsTable quedó sin uso; la fuente de verdad para sincronización es la cola pending_sync.

Decisión
- Eliminar la columna 'synced' de HabitsTable y HabitLogsTable.
- Subir schemaVersion a 3 y migrar usando ALTER TABLE para DROP COLUMN 'synced'.
- Renombrar/limpiar código que escribía en 'synced' y eliminar referencias en SyncService y datasources.
- Regenerar drift code con build_runner y actualizar tests.
- Documentar la decisión en ADR-009.

Consecuencias
- Menor superficie de mantenimiento; migración de DB necesaria; tests actualizados.
- PendingSync seguirá siendo la única fuente de verdad para la sincronización.

Verificación
- Run flutter pub get, flutter analyze y flutter test; verificar ausencia de referencias a 'synced'.
- Verificar schema version 3 y ausencia de columna 'synced' en las tablas afectadas.

Notas
- Este ADR complementa ADR-008 y ADR-007 al consolidar el modelo de datos y el flujo de sincronización.
